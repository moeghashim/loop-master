1 · Per-issue flow — solo vs reviewed

The dispatcher writes the cast at triage. CI is the floor; the reviewer is added only on the reviewed shape (a review: label). A red gate or an unapproved review sends work back to the executor; you merge.

2 · Sub-loop — the block relationship

An executor can spawn a child issue and pause on it. The child runs its own full loop; on its merge an Action clears the parent's block, and the parent resumes.

  P["Parent loop
executing"]:::agent --> B["Parent · blocked

  B -->|"spawn sub-issue"| C["Sub-loop #child
own interview + gates"]:::sub
  C -->|"merged → unblock (Action)"| RS["Parent · resumes"]:::agent
  RS --> MG["Parent merged
→ Done"]:::human
