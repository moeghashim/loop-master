§01 · THE PRINCIPLE

Mechanical truth, not agent claims.

An agent saying “it works” is a claim. A passing build is a fact. The entire system is built so that only facts turn things green — every claim shows as unverified until a mechanical signal confirms it.

executor · claim
“It works — I tested it.”
UNVERIFIED — ignored
vs
dispatcher · fact

  ✓ 214 tests
→ exit 0
VERIFIED — green

§03 · THE DISPATCHER

One process beats four pollers.

If every agent polls GitHub and self-selects work, you get claim races, double-writes, and locks. Instead, one deterministic process owns the loop: it polls, finds each issue’s next due step, invokes the right agent, runs the gate, and advances a labeled state machine.

one writer
no claim race, no lock — state changes through a single hand.
deterministic
same board state → same next step, every poll.
no agent ↔ agent
agents never talk to each other — only to the dispatcher.
gate-owned
the dispatcher runs make verify, never the agent.
dispatcher.toml 
poll_interval   = 20s
roster          = [codex, claude, pi, amp, cursor, factory]
roles           = [planner, executor, reviewer]
gate            = "make verify"
writer          = single
round_trip_cap  = 3
explore_rate    = 0.15
grader          = "02:00 nightly"
§04 · THE LEARNING LOOP

The cast gets smarter every night.

A blinded grader scores merged work from durable signals and a rubric. Scores roll into ratings, ratings drive the routing table, and the next cast exploits best-fit — sometimes exploring. A bandit over your roster.
